## 1. 产品背景与目标

### 1.1 现状（Step 2.5 之后）

Step 2.5 的只读 SVG 图谱验证了一个关键假设：**用户确实需要看 Agent 之间的关系图**。数据表明图谱页面的 PV 达到日均 10+，跳出率 < 60%，说明用户在使用它理解团队结构。

但管理关系仍然要靠 API 或 Agent 详情页逐个操作。当 Agent 数量增长到 10+ 时，关系管理的痛点开始显现：

- 管理员想调整组织架构，需要点 10 次 Agent 详情页逐条修改
- "Alice 和 Bob 之间现在是什么关系？"需要在图谱和详情页之间来回切换
- 添加新 Agent 后，设置它的汇报、协作、升级关系需要多处操作

### 1.2 目标

将只读图谱升级为**可拖拽编辑的交互式关系画布**：

1. **拖拽创建关系**：从 A 节点拖拽连线到 B 节点，选择关系类型，完成创建
2. **拖拽删除关系**：选中边后删除，或拖拽断开
3. **边类型切换**：选中已有的边，切换关系类型
4. **视觉区分增强**：4 种关系用视觉上明显不同的表现（实线竖/横、虚线、双实线+颜色）
5. **实时同步**：WebSocket 推送变更，多人同时查看图谱时保持同步

### 1.3 Step 5 完成后的协作体验

```
管理员进入关系图谱页面:

1. 看到 15 个 Agent 的组织图
   -> reports_to 形成清晰的层级树
   -> delegates_to / collaborates_with 水平分布
   -> escalates_to 红色双线醒目显示

2. 新 Agent @Eve 刚创建，还没关系
   -> Eve 节点浮动在画布空白处，高亮提示"未连接"

3. 管理员拖拽 Eve 连线到 Bob
   -> 弹出选择器: [reports_to] [delegates_to] [collaborates_with] [escalates_to]
   -> 选择 reports_to
   -> 关系即时创建，API 同步

4. 另一个管理员在另一台机器上打开图谱
   -> WebSocket 推送更新，Eve -> Bob 的边实时出现
   -> 无需刷新页面

5. 管理员选中 Alice -> Carol 的边
   -> 边高亮，弹出属性面板
   -> 可修改: 关系类型 / 权重 / 解除关系
```

## 3. 功能详述

### 3.1 技术选型

引入 `@xyflow/react`（ReactFlow v12）：

- 成熟的 React 流程图库，支持拖拽节点、连线、自定义边类型
- 内置 minimap、自动布局扩展
- 社区活跃，维护良好
- 替代 Step 2.5 的纯 SVG 实现（约 150 行 -> ReactFlow 组件化）

### 3.2 交互设计

#### 3.2.1 创建关系

```
操作: 从节点 A 的右下角锚点(handle)拖出连线到节点 B

流程:
1. 鼠标悬停在 Agent 节点上 -> 边缘出现 4 个锚点(上下左右)
2. 从锚点拖出一条虚线跟随鼠标
3. 拖到目标节点 B 上 -> 目标节点高亮
4. 松开鼠标 -> 弹出关系类型选择器

┌──────────────────┐
│ 选择关系类型:     │
│                  │
│ ◉ reports_to    │  <- 全局，Alice 汇报给 Bob
│ ○ delegates_to  │  <- 频道级
│ ○ collaborates  │  <- 频道级
│ ○ escalates_to  │  <- 全局
│                  │
│ 频道: [#frontend ▾]  <- 仅对 delegates_to / collaborates_with 显示
│                  │
│    [取消]  [确认] │
└──────────────────┘

5. 选择类型 + 频道（如需要）-> 确认
6. 边即时渲染在画布上
7. API POST /api/agent-relationships 同步持久化
```

#### 3.2.2 编辑和删除关系

```
选中边:
  - 单击边 -> 边高亮 + 弹出属性面板

属性面板:
┌──────────────────┐
│ 关系: reports_to │
│ 从: Alice        │
│ 到: Bob          │
│ 类型: reports_to │
│       [切换 ▾]   │  <- 下拉切换类型
│ 频道: (全局)     │
│ 权重: 1.0        │
│                  │
│ [解除此关系]     │  <- 红色按钮，二次确认
│ [关闭]           │
└──────────────────┘

删除方式:
  方式 1: 选中边 -> 属性面板 -> "解除此关系" -> 确认
  方式 2: 选中边 -> 按 Delete 键 -> 确认
  方式 3: 右键边 -> 上下文菜单 -> "解除关系"

切换类型:
  选中边 -> 属性面板 -> 切换下拉 -> 选择新类型 -> 边即时更新视觉
```

#### 3.2.3 节点交互

```
节点拖拽:
  - 拖拽节点到任意位置（手动调整 layout）
  - 拖拽时关联的边自动跟随

节点单击:
  - 跳转到 Agent 详情页（保留原行为）

节点右键:
  - 上下文菜单: [查看详情] [编辑关系] [聚焦此节点]
    - "聚焦此节点": 高亮该节点及其直接关联节点，其他节点淡化

节点状态指示:
  - 🟢 在线 (绿色圆点)
  - 🟡 离开 (黄色圆点)
  - 🔴 离线 (红色圆点)
  - 灰色 = 未连接任何关系（孤立节点）
```

#### 3.2.4 画布交互

```
Pan: 拖拽空白区域平移
Zoom: 滚轮缩放 (0.2x ~ 4x)，或左下角 +/- 按钮
自动布局: 按钮触发 dagre 自动排列节点
撤销/重做: Ctrl+Z / Ctrl+Shift+Z (本地操作历史栈)
快捷键:
  Delete     — 删除选中节点/边
  Escape     — 取消当前操作/关闭面板
  F           — 聚焦选中节点
  Ctrl+A      — 全选
  Ctrl+Z      — 撤销
  Ctrl+Shift+Z — 重做
```

### 3.3 视觉设计

#### 节点设计

```
┌───────────────────┐
│ 🟢  Alice         │  <- 在线状态圆点 + 名称
│     PM Agent      │  <- 角色/描述
│     #frontend     │  <- 主要频道（可选显示）
│     #backend      │
│   ─── ● ───       │  <- 底部锚点（上下左右 4 个）
│   ●       ●       │
│   ─── ● ───       │
└───────────────────┘
```

节点卡片轻微圆角 (8px)，白色底色，边框色 = Agent 在线状态色。选中节点时边框加粗 + 外发光。

#### 边设计（4 种关系视觉区分）

```

  ═══════════>  蓝色实线，带箭头的竖直线（纵向层级）
  颜色: #4A90D9
  粗细: 2px
  
delegates_to:
  ──────────>  绿色实线，带箭头的横直线（同级委托）
  颜色: #7ED321
  粗细: 2px

collaborates_with:
  - - - - - -> 橙色虚线，无箭头（无向关系显示为双向）
  颜色: #F5A623
  粗细: 1.5px
  虚线段: 8px 实 + 4px 空

escalates_to:
  ═══════════> 红色双实线，带箭头
  颜色: #D0021B
  粗细: 外线 2px + 内线 1px，间距 2px
  附带动画: 脉冲闪烁（可选，提示紧急）
```

### 3.4 WebSocket 实时同步

```
事件类型:
  - relationship:created  — 新关系创建
  - relationship:deleted  — 关系删除
  - relationship:updated  — 关系类型/权重变更
  - agent:online          — Agent 上下线状态变更
  - agent:updated         — Agent 名称/信息变更

行为:
  - 当前用户的操作: 乐观更新（即时渲染），API 成功后确认
  - 其他用户的操作: WebSocket 推送后增量更新图谱
  - 冲突处理: 后写覆盖，提示用户刷新（如检测到不一致）
  - 断线重连: 自动重连后全量同步一次
```

### 3.5 导出

```
导出格式:
  - PNG: 当前视图的截图（包含缩放级别）
  - SVG: 矢量格式（可缩放，适合文档嵌入）
  - solo export graph --format png --output org-chart.png

导出选项:
  - 包含/隐藏图例
  - 包含/隐藏离线节点
  - 白色/透明背景
```

## 4. 验收标准

**V1 — 拖拽创建关系**

```
GIVEN 关系图谱页面已加载，画布上有 Alice 和 Bob 两个节点
WHEN 管理员从 Alice 节点的锚点拖出连线到 Bob 节点
  AND 弹出关系类型选择器
  AND 管理员选择 reports_to 并确认
THEN Alice -> Bob 的蓝色实线竖线即时渲染在画布上
  AND POST /api/agent-relationships 被调用创建 reports_to 关系
  AND 关系持久化，刷新页面后仍存在
```

**V1b — 创建频道级关系**

```
GIVEN 管理员拖拽创建 delegates_to 关系
WHEN 在类型选择器中选择 delegates_to 并选择频道 #frontend
THEN 创建的关系 channel_id 为 #frontend
  AND 边为绿色实线横线
  AND 标签显示 "#frontend"
```

**V2 — 关系类型选择**

```
GIVEN 拖拽连线完成
WHEN 关系类型选择器弹出
THEN 4 种类型全部可选
  AND 全局类型 (reports_to/escalates_to) 不显示频道选择器
  AND 频道类型 (delegates_to/collaborates_with) 显示频道下拉选择器
```

**V3 — 选中边编辑**

```
GIVEN 画布上存在 Alice -> Bob 的 reports_to 边
WHEN 管理员单击该边
THEN 边高亮（颜色加深 + 外发光）
  AND 右侧弹出属性面板显示关系详情
WHEN 管理员点击 "解除此关系" 并确认
THEN 边从画布上消失
  AND DELETE /api/agent-relationships/:id 被调用
```

**V3b — 切换关系类型**

```
GIVEN 管理者选中一条 reports_to 边，打开属性面板
WHEN 将类型从 reports_to 切换为 delegates_to 并选择频道 #frontend
THEN 边视觉从蓝色实线竖线变为绿色实线横线
  AND API 更新关系类型
  AND 边标签显示 "#frontend"
```

**V4 — 视觉区分**

```
GIVEN 画布上存在 4 种关系类型各至少 1 条边
WHEN 用户查看画布
THEN 4 种边在"线型 + 颜色"上有明显差异，可一眼区分
  AND 即使灰度模式下也能区分（线型差异足够大）
```

**V5 — 自动布局**

```
GIVEN 画布上有 10 个 Agent 和 15 条关系，节点位置散乱
WHEN 用户点击 "自动布局" 按钮
THEN 节点按 dagre 层级布局重新排列
  AND reports_to 关系形成从上到下的层级
  AND 同一层级节点水平分布
  AND 没有节点重叠
```

**V5b — 手动拖拽节点**

```
GIVEN 用户拖拽节点 Alice 到新位置
WHEN 松开鼠标
THEN 节点保留在新位置
  AND 所有关联边自动更新路径
  AND 新位置存储到本地（localStorage），刷新页面后恢复
```

**V6 — WebSocket 实时同步**

```
GIVEN 用户 A 和用户 B 同时打开关系图谱
WHEN 用户 A 创建了一条新关系 Alice -> Bob
THEN 用户 B 的画布上实时出现该关系边（无需刷新）
  AND 没有明显的闪烁或跳变（增量更新）
```

**V7 — 导出**

```
GIVEN 画布上显示 10 个 Agent 和 15 条关系
WHEN 用户点击 "导出 PNG"
THEN 浏览器下载一张 PNG 图片
  AND 图片包含当前画布可见的所有节点和边
  AND 图片分辨率与画布一致
```

**V8 — 频道背景色区分**

```
GIVEN 画布显示 "全局" 视图，包含多条频道级关系
WHEN 用户查看
THEN 相同频道的节点和边有相同的淡色背景/分组标识
  AND 频道切换为 "仅 #frontend" 时，不相关节点和边变灰
```

## 5. 非功能需求

### 5.1 性能

| 指标 | 目标 |
|------|------|
| 图谱首次渲染（50 Agent, 200 边） | < 1s |
| 拖拽操作帧率 | >= 30fps |
| WebSocket 消息延迟 | < 200ms |
| 自动布局计算（50 节点） | < 500ms |
| 内存占用（50 Agent 图谱页） | < 50MB（含 ReactFlow 运行时） |

### 5.2 可用性

- 键盘导航覆盖所有常用操作
- 图例始终可见（固定在底部或可折叠）
- 新用户首次进入时显示引导提示："拖拽节点连线创建关系"
- 错误操作提供撤销（Undo/Redo 栈，支持 50 步）
- 符合 WCAG AA 标准（颜色不是唯一区分手段，线型也区分）

## 6. 范围边界

### 本次做

- ReactFlow 拖拽编辑器（创建/删除/编辑关系）
- 4 种关系的视觉区分（线型 + 颜色）
- 拖拽节点调整位置
- 自动布局（dagre）
- 属性面板（选中边后显示详情和编辑入口）
- WebSocket 实时同步
- Undo/Redo
- 导出 PNG/SVG
- 频道级关系的背景色分组
- 快捷键支持

### 本次不做

- 批量编辑关系（一次创建多条）
- 关系模板（预设团队结构一键套用）
- 图谱版本历史/回退
- 图谱嵌入其他页面（仅独立页面）
- 基于图谱的"影响分析"（如删除某 Agent 后自动提示受影响的关系链）
- 图谱导入（从外部 OrgChart 工具）

## 7. 依赖与风险

### 依赖

| 依赖项 | 说明 | 风险等级 |
|--------|------|----------|
| Step 1 Agent Relationships API | 图谱数据来源 | 低 — 已实现 |
| Step 2.5 只读图谱 | 验证了可视化价值，确定了节点和边的视觉基础 | 低 — 已实现 |
| WebSocket 基础设施 | 实时同步依赖 WebSocket 推送 | 中 — 需确认现有 WebSocket 方案 |
| @xyflow/react (ReactFlow v12) | 新增依赖 | 低 — 成熟库 |
| dagre | 自动布局算法 | 低 — MIT 许可 |

### 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Agent 数量 10+ 后才启动，但实际增长慢 | Step 5 迟迟不触发 | 不强制 10+ 门槛，Step 2.5 数据达标即可启动 |
| ReactFlow 复杂交互导致前期开发时间超预估 | 延期 | 分阶段交付：先实现拖拽创建+删除，后迭代自动布局+导出 |
| WebSocket 同步在高并发下丢消息 | 数据不一致 | 客户端定期（30s）全量 diff 校验；检测不一致时提示刷新 |
| 拖拽误操作创建错误关系 | 数据污染 | 撤销 + 删除时需要二次确认；操作日志记录 |
| 新增依赖 @xyflow/react 体积大 | 前端 bundle 增大 | 动态 import 懒加载图谱页面 |
